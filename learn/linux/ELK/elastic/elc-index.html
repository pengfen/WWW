索引 ---------------------------------------------------------------- 在Elasticsearch中存储数据的行为就叫做索引(indexing) 索引(名词) 一个索引(index)就像是传统关系数据库中的数据库 它是相关文档存储的地方 索引(动词) 索引一个文档 表示把一个文档存储到索引(名词)里 以便它可以被检索或者查询 这很像SQL中的INSERT关键字 差别是如果文档已经存在 新的文档将覆盖旧的文档 倒排索引 传统数据库为特定列增加一个索引 B-Tree索引来加速检索 Elasticsearch和Lucene使用一种叫做倒排索引(inverted index)的数据结构来达到相同目的 建立一个员工目录 (需求) ---------------------------------------------------------------- 数据能够包含多个值的标签 数字和纯文本 检索任何员工的所有信息 支持结构化搜索 (查找30岁以下的员工) 支持简单的全文搜索和更复杂的短语(phrease)搜索 高亮搜索结果中的关键字 能够利用图表管理分析这些数据 索引员工文档 ---------------------------------------------------------------- 存储员工数据 每个文档代表一个员工 在Elasticsearch中 文档归属于一种类型(type) 而这些类型存在于索引(index)中 与传统关系型数据库对比 Relational DB ---> Databases ---> Tables ---> Rows ---> Columns Elasticsearch ---> Indices ---> Types ---> Documents ---> Fields Elasticsearch集群可以包含多个索引(indices)(数据库) 每一个索引可以包含多个类型(types)(表) 每一个类型包含多个文档(documents)(行) 每个文档包含多个字段(Fields)(列) 创建员工目录 ---------------------------------------------------------------- 为每个员工的文档(document)建立索引 每个文档包含了相应员工的所有信息 每个文档的类型的为employee emplyee类型归属于索引 eauths eauths 索引存储在Elasticsearch集群中 PUT /eauths/employee/1 {"first_name" : "cao", "last_name" : "peng", "age" : 28, "about" : "welcome to kibana", "interests" : ["sports" : "music"]} eauths 索引名 employee 类型名 1 员工ID 使用x-pack添加数据 curl -XPUT -u elastic http://192.168.233.132:9200/eauths/employee/1 -d '{"first_name" : "cao", "last_name" : "peng", "age" : 28, "about" : "welcome to kibana", "interests" : ["sports", "music"]}' {"_index":"eauths","_type":"employee","_id":"1","_version":1,"result":"created","_shards":{"total":2,"successful":1,"failed":0},"created":true} #
